**The-Hands-On-Approach-Active-Learning-The-80-to-20 Rule** The 80/20 Rule is a highly effective strategy for mastering programming. 
It shifts your focus from passive consumption to active creation.

### Why Students Should Use It
- Instead of mindlessly copying an instructor, pausing to solve the logic yourself builds genuine problem-solving skills.
- Modifying and intentionally breaking existing code helps you understand why things work, not just that they work.
- Spending 80% of your time coding ensures you are actually learning by doing, rather than forgetting what you read.

### How to Get Started
1. Apply the 80/20 Balance: Limit your reading or video-watching to 20% of your study time, and spend the remaining 80% actually writing code.
2. When watching a coding tutorial, pause the video before the instructor types the solution and try to write the logic yourself.

  ## Smart-Debugging-Trace-your-Variables(-Debugging-Trace-your-Variables) 
**smart debugging** Refers to using modern intergrated development environment tools to pause code execution,inspect data onthe flyand automate log colections.  
**how to trace your variables** is the process of monitoring what a variable's value is at spesific point in your codes's execution.

### Why Students Should learn to Use the technique
- it helps you identify the break points. A marker you set on a spesific line of code.
- Awarness of the watch windows. you can manualy list spesif variabeles to follow
- If your variabel is being passed around the call stack shows you the exact sequence of function calls that led to the variabel current.
- It also teaches conditional breakpoint. A type of breakpoint that dosent pause a program.

### How to Get Started
1. Leverage watch expressions and variables inspection.
2. Set conditional breakpoints.
3. Trace the call stack.
